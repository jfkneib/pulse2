Notes sur l'installation

Configuration système requise :

  • Microsoft Windows 98, ME, NT (Service Pack 6 ou version ultérieure), 2000 ou XP
  • PC compatible Pentium
  • 64 Mo de RAM
  • 250 Mo (version CJK : 300 Mo) d'espace libre sur le disque dur
  • Résolution d'écran de 800x600 minimum, au moins 256 couleurs

Remarque : il est impossible de procéder à l'installation sans posséder les droits d'administrateur requis.
Remarque à l'intention des utilisateurs de Windows 98 : si vous installez Java au cours du processus d'installation d'OpenOffice.org, le programme d'installation vous demandera de redémarrer le système. Vous pouvez ignorer ce message. Si vous choisissez de redémarrer le système, relancez ensuite le programme d'installation d'OpenOffice.org.

Les pilotes des cartes graphiques sont souvent la cause de certains problèmes de démarrage d'OpenOffice.org (par ex. blocage) ou d'affichage à l'écran. Si vous rencontrez ce genre de situations, mettez à jour le pilote de votre carte graphique ou faites un test avec le pilote graphique standard du système d'exploitation. Les problèmes d'affichage des objets 3D peuvent souvent être résolus en désactivant l'option "Utiliser Open GL" sous Outils - Options - Affichage - Affichage 3D.

Vous pouvez installer OpenOffice.org 2.3 même si une version plus ancienne d'OpenOffice.org est déjà installée. Si vous souhaitez désinstaller l'ancienne version par la suite, il vous faudra lancer le programme d'installation de la nouvelle version et choisir Réparer. Cela garantit l'enregistrement correct de la nouvelle version sur votre système.
Les opérations de copier-coller réalisées via le presse-papiers d'OpenOffice.org 2.0 vers OpenOffice.org 2.0 risquent de ne pas s'exécuter si le format OpenOffice.org est sélectionné. Dans ce cas, sélectionnez Édition - Collage spécial, puis choisissez un autre format, ou ouvrez le document directement dans OpenOffice.org 2.0.

Vérifiez qu'il y a suffisamment d'espace libre dans le répertoire temporaire de votre système et que vous disposez des droits d'accès en lecture, écriture et exécution pour ce répertoire. Avant de lancer l'installation, fermez tous les autres programmes.

Installation réseau : Erreur lors du lancement du démarrage rapide d'OpenOffice.org

Si, au démarrage du système, le lancement du démarrage rapide d'OpenOffice.org ne s'exécute pas correctement (ex. : lien non valide), le lecteur réseau contenant l'installation réseau n'est probablement pas disponible.

Ces problèmes interviennent principalement lorsque les lecteurs réseau sont connectés via un script dédié. Ainsi, la connexion des lecteurs s'effectue en parallèle à l'exécution des programmes du groupe de démarrage automatique à partir duquel le démarrage rapide est également lancé.

Pour éviter ces situations, il faut connecter le lecteur réseau via les services du système. En cas d'impossibilité, supprimez du groupe de démarrage automatique du système le lien utilisé pour lancer le démarrage rapide, ou désactivez le démarrage rapide lors de l'installation de la station de travail. Le démarrage rapide peut aussi être désactivé par l'administrateur système au moment de l'installation réseau. Dans ce cas, l'utilisateur ne pourra pas sélectionner le démarrage rapide lors de l'installation ultérieure de la station de travail.

ALPS/Synaptics notebook touchpads in Windows

Due to a Windows driver issue, you cannot scroll through OpenOffice.org documents when you slide your finger across an ALPS/Synaptics touchpad.

To enable touchpad scrolling, add the following lines to the "C:\Program Files\Synaptics\SynTP\SynTPEnh.ini" configuration file, and restart your computer:

[OpenOffice.org]FC = "SALFRAME"SF = 0x10000000SF |= 0x00004000

Note: The location of the configuration file might vary on different versions of Windows.

Ai Squared ZoomText 7.11

Si vous souhaitez utiliser Ai Squared ZoomText avec OpenOffice.org 2.3, vous devez être équipé de la version 7.11 minimum. Seules les versions de Ai Squared ZoomText téléchargées après le 12 juin 2002 offrent les fonctionnalités requises.

Modification de la police de l'interface utilisateur

Dans OpenOffice.org, la police utilisée pour l'affichage à l'écran et les impressions peut être remplacée par toute autre police installée sur le système, via la fonction de remplacement de police. Choisissez Outils - Options - OpenOffice.org - Polices pour accéder à la table de remplacement de la police.

Pour modifier la police de l'interface utilisateur d'OpenOffice.org, remplacez la police par défaut, Andale Sans UI, par une autre police, puis activez l'option Toujours.

Pour obtenir des informations détaillées sur la boîte de dialogue, reportez-vous à l'aide d'OpenOffice.org.

Raccourcis clavier

Vous ne pouvez employer dans OpenOffice.org que des raccourcis clavier (combinaisons de touches) non utilisés par le système d'exploitation. Si une combinaison de touches dans OpenOffice.org ne fonctionne pas selon la description de l'aide d'OpenOffice.org, il est possible que ce raccourci soit déjà utilisé par le système d'exploitation. Pour résoudre de tels conflits, modifiez par exemple les raccourcis définis par votre système d'exploitation. Vous pouvez aussi modifier la plupart des raccourcis définis dans OpenOffice.org. Pour plus d'informations à ce sujet, reportez-vous à l'aide d'OpenOffice.org ou à l'aide de votre système d'exploitation.

Problèmes lors de l'envoi de documents par e-mails à partir de OpenOffice.org

Il est possible que le programme s'arrête brutalement ou se bloque lorsque vous tentez d'envoyer un document par e-mail en choisissant Fichier - Envoyer - Document par e-mail ou Document comme fichier PDF joint. En effet, le fichier système Windows MAPI (Messaging Application Programming Interface) génère des erreurs avec certaines versions de fichiers. Malheureusement, il est impossible de réduire ce problème à un certain nombre de versions. Pour de plus amples informations, consultez la page http://www.microsoft.com, puis tapez "mapi dll" dans le champ de recherche de la Base de connaissances Microsoft.

Nous vous remercions de bien vouloir suivre la procédure minimale d'enregistrement du produit lors de l'installation du logiciel. L'enregistrement est facultatif, mais nous vous serions reconnaissants de prendre quelques minutes pour l'effectuer, car les informations fournies aideraient la communauté dans son effort d'amélioration de la suite logicielle et de réponse directe aux besoins des utilisateurs. Pour protéger vos données personnelles, la communauté OpenOffice.org applique une politique de confidentialité stricte. Si vous n'avez pas suivi la procédure d'enregistrement au moment de l'installation du produit, vous pouvez l'effectuer à tout moment en consultant la page www.openoffice.org/welcome/registration-site.html

Participation à des projets

Vous pouvez apporter une contribution précieuse à ce grand projet Open Source quelle que soit votre expérience en programmation et conception logicielle. Oui, vous !

À l'adresse http://projects.openoffice.org/index.html vous trouverez des projets de localisation, de portage, de travail collaboratif et aussi des projets de programmation. Si vous n'êtes pas développeur, vous pouvez participer au projet de marketing et documentation. Le projet marketing OpenOffice.org applique à la fois des techniques commerciales classiques et d'autres techniques plus agressives pour positionner le logiciel Open Source sur le marché. Comme nous nous attaquons à des zones variées, culturellement et linguistiquement, vous pouvez aussi nous aider tout simplement en parlant autour de vous de cette suite bureautique.

Vous pouvez nous aider en rejoignant le réseau d'informations et de communications marketing à l'adresse suivante : http://marketing.openoffice.org/contacts.html Vous pouvez fournir des références de contact avec la presse, les médias, les organismes publics, les cabinets de conseil, les écoles, les groupes d'utilisateurs de Linux et les développeurs dans votre pays et votre région.

Support utilisateur

Pour obtenir de l'aide sur la suite bureautique OpenOffice.org 2.0, consultez les archives où vous trouverez des réponses aux questions posées précédemment dans la liste de diffusion users@openoffice.org à l'adresse www.openoffice.org/mail_list.html. Si vous préférez, publiez vos questions sur la liste users@openoffice.org. N'oubliez pas de vous inscrire à la liste de diffusion afin de recevoir une réponse par e-mail.

La section FAQ est également à votre disposition dans la barre de navigation, à gauche sur la page d'accueil d'OpenOffice.org. Les questions les plus courantes se trouvent dans cette Foire aux questions.http://user-faq.openoffice.org/.

Comment commencer

Pour commencer à contribuer, inscrivez-vous aux listes de diffusion de votre choix, observez un moment puis, peu à peu, utilisez les archives pour vous familiariser avec les nombreux sujets abordés depuis octobre 2000, date à laquelle le code source OpenOffice.org a été diffusé. Dès que vous vous sentez à l'aise, envoyez un e-mail pour vous présenter aux autres membres et participez. Si vous connaissez bien les projets Open Source, consultez la liste des tâches : peut-être souhaiterez-vous participer à l'une d'elles http://development.openoffice.org/todo.html.

Nous vous souhaitons un agréable travail avec OpenOffice.org 2.3 et espérons vous rencontrer prochainement sur le site Web.
